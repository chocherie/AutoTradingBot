# Rule: Read Spec Before Coding

Before implementing or modifying any module, read the relevant specification in `specs/`:

- Data collection → `specs/data-pipeline.md`
- Claude integration → `specs/claude-brain.md`
- Portfolio/positions → `specs/portfolio-engine.md`
- Risk management → `specs/risk-management.md`
- Web dashboard → `specs/web-dashboard.md`
- Full system → `specs/system.md`

The spec is the source of truth. If the code diverges from the spec, either fix the code or update the spec with justification.
